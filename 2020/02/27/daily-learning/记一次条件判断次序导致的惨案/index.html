记一次条件判断次序导致的惨案

发表于 2020-02-27 更新于 2020-11-23 分类于 日常学习

起因

leetcode第23题。

合并 k 个排序链表，返回合并后的排序链表。请分析和描述算法的复杂度。

这个问题几乎直接套用mergesort递归的代码就可以了。给出我的实现：

看题解中有另一种递归思路，就是每个结点每个结点的递归，和上面那个merge代码的递归思路很像，不过就是拓展到了n个链表而已。于是我便开始写bug了…

碰到的问题

代码也很简单，不过我是用的C语言，因此不能直接将已经为空的链表从arraylist那种链表中直接remove，所以多了中间几行把非空的链表换到已经成空链表的位置，并且使listsSize减一。
实际上这个代码对于一部分case是有问题的，后来通过一些方法解决了，但是不是讨论的重点就不赘述了。
最初版本for循环里面的条件是i < listsSize，后来测试发现如果lists中间如果有空链表那么会发生错误，对于非空的情况运行良好。因为当时误以为所有非空的链表都排在空链表的前面，所以我就改成了lists[i] != NULL && i < listsSize，乍一看好像没有什么问题，不过测试发现对于非空链表的case也过不了了。错误一直是堆溢出。反复检查代码后始终没有看出明显错误。最后定位到for循环里面出了问题才发现我的这个判断条件会导致bug…
具体的问题其实是条件判断的顺序是从左到右的，这里先判断lists[i] != NULL，很显然当i = listsSize - 1且循环结束了过后i++导致i = ListsSize，这里就越界了。
最开始写代码的时候会犯这种错误，结果写了这么久代码仍然会犯这种错误…浪费总时长不少于5个小时，太亏了。

总结

  • 对于是否越界的判断始记终放在第一个。

C语言程序设计现代方法

两天把C语言程序设计现代方法过了一遍，总结一下。

C语言概述

这章没什么好说的，唯一觉得比较惊艳的就是gcc里面的-Wall选项，这个选项可以更清楚的给出程序中可能存在的问题。更好一点的检查工具是lint，但是目前并不是特别想管这些。

C语言基本概念

编译和链接

C语言源代码转化成机器指令的三个步骤：

  • 预处理。把定义的宏以及include的内容直接搞过来。
  • 编译。编译器把预处理过的程序翻译成机器指令，生成.o结尾的文件。但是还不能运行。
  • 链接。链接器（linker）把编译器产生的目标代码和其他附加代码整合到一起，产生可以执行的程序。

一般以#开头的称为指令。函数里面叫语句。

变量和赋值

赋值的右侧应该是一个表达式（expression）。

标识符

标识符可以含有字母、数字、下划线。但是只能以字母或下划线开头。
C语言区分大小写。
不能用关键字作标识符。

格式化的输入/输出

讲了讲printf，scanf，提了一下gets和puts。

字符串有个性质就是int str = "a1" "a2";这里得到的str是"a1a2"。也就是说拼接两个字符串常量的工作可以直接交给编译器去做。

表达式

关键是优先级。

算数运算符

包括：

  • 一元运算符。只要求一个操作数。如：+、-。int a = -b;
  • 二元运算符。要求两个操作数。如：+、-、*、/、%。

由实现定义（implementation-defined）：由具体的平台的实现决定。

二元运算符是左结合的（从左往右运算），一元运算符是右结合的（从右往左，如+-i = +(-i)）。

赋值运算符

赋值实际上也是一个运算符。因此可以写出这样的代码while(c=*str++)用于遍历一个字符串。

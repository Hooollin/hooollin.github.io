引言

冯诺依曼体系结构

冯诺依曼设计计算机的主要思想：

  1. 用二进制的代码表示程序和数据；
  2. 计算机采用存储程序的工作方式；
  3. 计算机硬件由存储器、运算器、控制器、输入设备和输出设备等五大部件组成。

冯诺依曼体系结构：

  1. 包括了Central Processing Unit单元：算数逻辑单元、处理器寄存器、控制单元
  2. 包括了存储器（数据和指令）
  3. 输入输出机制

冯诺依曼体系结构的两个核心是数据和对数据的操作。

哈弗体系结构

哈弗体系结构特点：

  1. 存储程序系统
  2. 有独立的数据存储区和数据总线
  3. 有独立的指令存储区和指令总线

基于哈弗体系结构的典型处理器有DSP。

冯诺依曼vs哈弗

相同点：

  • 都采用了存储程序的工作方式

不同点：

  • 冯诺依曼体系结构中存储器不仅存放指令，还存放数据；哈弗体系结构中有独立的数据、指令存储器和数据、指令总线。

计算机的分类

Flynn’s 分类：基于指令流和数据流的计算机结构分类

指令流：机器执行的指令序列。
数据流：由指令流调用的数据序列， 包括输入数据和中间结构，不包括输出数据。
多倍性：在系统性能瓶颈上同时处于同一执行阶段的指令或数据的最大可能个数。

  • SISD(Single Instruction Single Data)
    • Uniprocessors
    • 传统的顺序执行的单处理器计算机，其指令部件每次只对一条指令进行译码，并只对一个操作部件分配数据。冯诺依曼架构。
  • MISD(Multiple Instruction Single Data)
    • ???
  • SIMD(Single Instruction Multiple Data)
    • IIIiac-IV, CM-2
    • Simple programming model
    • Low overhead
    • Flexibility
    • All custom
    • 并行处理机（阵列处理机）为代表，并行处理机包括多个重复的处理单元，由单一指令部件控制，按照同一指令流的要求为他们分配各自所需要的不同数据。
  • MIMD(Multpiple instruction Multiple Data)
    • SPARCCenter, T3D
    • Flexible
    • Use off-the-shelf micros
    • 实现作业、任务、指令等各级全面并行的多机系统。多处理机属于MIMD。无论是大规模并行处理机还是对称多处理机，都属于MIMD。

计算机系统结构定义与计算机的设计任务

计算机系统结构的原始概念：由程序员（机器语言）看见的（计算）系统属性，即概念性结构和功能行为，以及区分数据流动和控制逻辑设计的组成及物理实现。

Instruction Set Architecture：硬件和软件之间的接口。

指令集结构设计

一个好的接口需要具备的特点：

  • 允许有多种实现（portability, compatability）——对于指令集本身来说
  • 用在很多不同的方面（generality）——对于指令集本身来说
  • 为更高层提供方便的功能——指令集为更高层提供的服务
  • 允许更底层能有效实现——指令集在下层中的实现

ISA的七个重要要素：

  • ISA的类型：现代通用寄存器结构，早期累加器结构
  • 存储器的访问方式：按字节访问或按字访问
  • 寻址方式
  • 操作数的类型和大小：8位字符，32位整型数
  • 操作类型：数据传输，算术、逻辑运算
  • 控制流指令：转移，子程序调用/返回
  • ISA编码：固定长度，可变长度

计算机系统结构：满足功能、性能和价格目标的条件下，设计、选择和互连硬件部件构成计算机。

系统结构包括：

  • 指令系统设计
  • 组成（Organization）：计算机设计方面的高层次
    • CPU内部结构、存储器、I/O系统、多处理器、网络
  • 硬件：计算机的具体实现技术
    • 详细逻辑设计、封装、冷却系统、工号

影响计算机系统结构的因素：

  • 实现技术
  • 应用程序
  • 编程语言
  • 编译器
  • 操作系统
  • …

集成电路功耗的趋势

从第一个微处理器的1Hz/10watt到现在的2GHz/135watt，功耗对芯片的规模也提出了挑战
技术挑战：

  • 分配功率
  • 散热
  • 避免过热点

动态功率：开关晶体管产生的功耗

  • $power_dynamic=\frac{1}{2} * Capacitive\quad load * Voltage^2 * Frequency\quad switched$
  • $Energy_dynamic=Capacitive\quad load * Voltage^2$
    静态功率：晶体管在关闭时漏电产生的功耗
  • $Power_static=current\quad static*Voltage$

经验法则：电压减少10%->频率减少10%，功率减少30%，性能减少小于10%

对于多核，每瓦提供更多性能

假设一个small core的功率为1，性能为1，那么一个big core的功率为4，性能为2，4个small core的功率为4，性能为4，可以得出：

  • 功耗与面积成正比
  • 单线程性能与面积增加倍数的一半成正比
  • 多核有更高的功率利用率

可靠性

可靠性：广义上包括可靠性、安全性和可用性

计算机系统的可靠性：用于表示系统提供给用户服务的质量，这里”可靠性“可用”信任“代替。（dependability $=$ reliance）

模块可靠性：从模块可用到出现故障的持续服务度量

  • MTTF(mean time to failure)：平均无故障时间，$\frac{1}{FIT}$
  • MTTR(mean time to repair)：平均修复时间
  • FIT(故障率)：$\frac{1}{MTTF}$
  • MTBF(平均故障间隔时间) = MTTF + MTTR

模块可用性

  • $\frac{MTTF}{MTTF + MTTR} = \frac{MTTF}{MTBF}$

提高可靠性的方法：

  • 冗余（Redundancy）：
    • 时间冗余（重复操作直到无错）
    • 资源冗余（配置另外的相同部件，有错时用于替代出错部件）

测量、报告和总结计算机性能

计算机性能的定义：

  • 响应时间：Response time，针对单任务
    • 从提出请求到被执行的时间
  • 吞吐率：Throughput，针对数据中心
    • 单位时间内所做的工作

工业性能指标：MIPS

  • MIPS-Millions of Instructions per Second
  • 用相同的指令集比较两台机器的MIPS一般是公平的
  • MIPS可能是一个”无意义的性能指标”
  • $MIPS = \frac{\frac{number\quad of \quad instructions}{benchmark} * \frac{benchmark}{total\quad runtime}}{1,000,000}=\frac{number\quad of\quad instructions}{totalruntime * 1,000,000,000}$

CPU时间：执行某一任务在CPU的花费的时间

时钟周期：Clock cycle，$\frac{1}{时钟频率}$

计算公式：$CPU时间=CPU时钟周期数时钟周期=\frac{CPU时钟周期数}{时钟频率}$，$CPU时钟周期数=时钟频率CPU时间$

改进行性能的方法：

  • 减少时钟周期数；
  • 增加时钟频率（减少时钟周期）

计算机A：2GHz, CPU时间10秒
设计计算机B：

  • 目标Cpu时间缩短为6秒
  • 方法提高时钟频率，但是时钟周期数会变为计算机A的1.2倍
    问计算机2的时钟频率应该为多少

设时钟频率为$\sigma$，则$时钟周期数=CPU时间_B\times \sigma = 1.2\times 2\times 10^9 \times CPU时间_A$，解得$\sigma=4GHz$

CPI (Clock cycle per Instruction)：表示执行每条指令所需的平均时钟周期数，是一个程序全部指令所用时钟周期数的平均值。
$$CPU时钟周期数=程序指令数\times CPI$$
$$CPU时间=程序指令数\times CPI \times 时钟周期 =\frac{程序指令数\times CPI}{时钟频率}$$

计算机A：时钟周期250ps,CPI=2.0
计算机B：时钟周期=500ps,CPI=1.2
两台计算机的ISA相同，实现方式不同，问哪个计算机更快

$CPU时间_A=程序指令数\times 250ps \times CPI_A=程序指令数 * 500ps$，
$CPU时间_B=程序指令数\times 500ps \times CPI_B=程序指令数*600ps$，$A更快，快20%。$

比较两台计算机性能时应考虑的三个因素：

  • 指令数：使用仿真器软件工具可以测量出
  • CPI：与计算机各种设计细节密切相关，以及应用程序中不同类型指令所占的比例。
  • 时钟频率

只用一种因素去评价性能是危险的。当比较两台计算机时，必须考虑全部三个因素。
唯一能被完全可靠测量的计算机性能指标是时间。
对指令集减少指令数的改进可能降低时钟周期或提高CPI，抵消了改进的效果。
指令数最少的代码其执行速度未必是最快的。

不同类型的基准测试程序（benchmarks）

  • 核心测试程序
  • 小程序测试
  • 综合测试程序
  • 基准测试程序集：选择一组有代表性的不同类型应用程序，集中起来构成基准测试程序集，以有效评测计算机处理各种应用的性能。

最好的基准测试程序就是实际应用程序，因为他们反应了终端用户的需要。

SPEC - The System Performance Evaluation Cooperative

计算机性能通常峰值性能（Peak performance）和持续性能（Sustained performance）来评价

  • 峰值性能是指在理想状况下计算机系统可获得的最高理论性能，他不能反映出系统的实际性能。
  • 实际性能又称持续性能，其值只有峰值性能的5%~35%，这是应为实际程序运行时会受到硬件结构、操作系统、算法设计和变成等因素的影响。

计算机设计的量化原则

  • 利用并行性
  • 局部性原理
  • 注重经常性时间
  • Amdahl’s law
  • CPU性能公式

利用并行性

改善计算机性能最重要的方法。
并行性的层次：

  • 线程级或任务级：使用多个处理器，GPU
  • 指令级：流水线、超标量、OOO?
  • 操作级：并行加法器、组相联cache、功能部件流水线

局部性原理

程序特性：趋向于重用最近用过的数据和指令。一个程序90%的执行时间仅仅执行其10%的代码。

  • 时间局部性: 最近访问过的项很可能近期将被访问
  • 空间局部性：地址相近单元的内容趋向于在一定时间内被相近访问。

注重常用事件

量化这个原则的基本定律：Amdahl’s Law。
采用更快的执行方式后所获得的系统性能提高，与这种执行方式的使用频率或占总执行时间的比例有关。

Amdahl’s Law：系统中某一部件由于采用了更快的执行方式后所获得的系统性能的提高，于这种执行方式的使用频率或占总执行时间的比例有关。
amdahl’s law定义了加速比：
$$加速比=\frac{采用改进措施后计算机的性能}{没有采用改进措施时的计算机的性能}=\frac{没有采用改进措施时某任务的执行时间}{采用改进措施后某任务的执行时间}$$

和加速比有关的两个因素：

  • 能被改进的部分在总执行时间中所占的比例$Fe=\frac{可改进部分的执行时间}{改进前整个任务的执行时间}$
  • 改进部分采用改进措施后，相比没有采用改进措施前性能提高的倍数$Se=\frac{改进前改进部分的执行时间}{改进后改进部分的执行时间}$，称为改进加速比。

于是我们可以得出如下结论：

  1. 改进后整个任务的执行时间$Tn=T_0(1 - Fe + \frac{Fe}{Se})$，不可改进时间加上可改进部分时间除以改进加速比。
  2. 改进后整个系统的加速比为：$S_n=\frac{T_0}{T_n}=\frac{1}{(1-Fe) +\frac{Fe}{Se}}$

根据系统加速比的公式，可以得出如下结论：

  • 当$F0$为0时，没有可以改进的部分，$S_n$为1。
  • 当$Fe$为1时，$S_n=S_e$，性能的提高幅度等于改进后性能提高的倍数。若改进前是单核，改进后是4核，则加速比为4。
  • 当$S_e->\inf$，$S_n=\frac{1}{1-Fe}$，可获取性能改善的极限受$Fe$的约束。

1443D Extreme Subtraction

发表于 2020-11-25 分类于 competitive programming

题面

You are given an array a of n positive integers.

You can use the following operation as many times as you like: select any integer 1≤k≤n and do one of two things:

  • decrement by one k of the first elements of the array.
  • decrement by one k of the last elements of the array.

For example, if $n=5$ and $a=[3,2,2,1,4]$, then you can apply one of the following operations to it (not all possible options are listed below):

  • decrement from the first two elements of the array. After this operation $a=[2,1,2,1,4]$;
  • decrement from the last three elements of the array. After this operation $a=[3,2,1,0,3]$;
  • decrement from the first five elements of the array. After this operation $a=[2,1,1,0,3]$;

Determine if it is possible to make all the elements of the array equal to zero by applying a certain number of operations.

简单翻译一下就是给定一个数组，可以从把这个数组中前面$k$个数都减$1$，或者把这个数组中的后面$k$个数减$1$，问最后能不能使整个数组中的元素都等于$0$。

思路

首先考虑只能把这个数组后面的$k$个数减$1$，那么这个数组必须满足：
$$a[i + 1] >= a[i]$$
因为很显然，如果$a[i + 1] < a[i]$，当$a[i + 1]$被减到$0$时，$a[i] > 0$，此时已经无法将$a[i]$减到$0$。
得到这个重要的性质后，现在考虑可以把数组的前面或后面$k$个数都减$1$都情况。
我们知道如果这个数组非递减的，那么一定可以将所有的元素都变为0。对于数组并不是非递减的情况，其中的某个元素$a_i$，有$a_{i + 1} < a_i$，那么我们已经无法从数组后面将$a_i$变为0，只能考虑从数组前面把$a[i]$变为0。
那么$a[i]$什么情况下可以从前面变为0呢？不难发现是否能从前面进行修改，使得数组满足$a[i + 1] >= a[i]$这条性质，取决于$a[i]$还可以减少的数量$k$。如果有$k \geq a[i] - a[i + 1]$，那么我们可以从前面将$a[i]$减小至$a[i+1]$，使数组满足$a[i + 1] >= a[i]$这条性质。如果$k < a[i] - a[i + 1]$，那么这个数组中的元素就无法都变为0。

放球模型

发表于 2020-11-23 分类于 课程

放球模型

以下所有放球模型，未作特殊说明，球的数量为$n$, 盒子的数量为$m$。

球无区别，盒子无区别（允许有空盒）

整数的拆分，方案数$p_d(n)$。

球无区别，盒子无区别（不允许有空盒）

显然当$n < m$时为0。当$n \geq m$时，现在每个盒子里面都放一个球，问题转化为球的个数为$n - m$，盒子的个数为$m$，球无区别，盒子无区别同时允许有空盒的方案数，对应于整数的拆分，方案数为$p_d(n - m)$。

球有区别，盒子无区别（不允许有空盒）

  • 当$m > n$时，可行的方案书显然为0，因为一定有空盒存在。
  • 当$m \leq n$时，将m个小球分成m个集合，且无一空集的方案数。对应的模型为第二类stirling数，方案数为$S_2(n, m)$。

第二类stirling数的性质:

  1. $S_2(n + 1, k) = S_2(n, k - 1) + k * S_2(n, k)$
  2. $S_2(n, n - 1)$ = $(^n_2)$
  3. $S_2(n, 2) = 2^{n - 1} - 1$
  4. $n < m, s_2(n, m) = 0$

球有区别，盒子无区别（允许有空盒）

结论：$S_2(n, 1) + S_2(n, 2) + … + S_2(n, n - 1) + S_2(n, n)$。

和第二类stirling数的区别就在于允许有空盒。空盒的数量可以为$0, 1, 2,…,m - 1, m$。当空盒的数量为$k$时，剩余的空盒数量为$m - k$，对应的模型为第二类stirling数，$S_2(n, m - k)$。容易得出方案数为$S_2(n, 1) + S_2(n, 2) + … + S_2(n, n - 1) + S_2(n, n)$。

球无区别，盒子有区别（允许有空盒）

结论：方案数为$F(m, n) = C(m + n - 1, n)$。
隔板法：$m$个盒子需要$m - 1$个隔板，总共有$m - 1 + n$个位置，从中选出$m - 1$个位置作为隔板或者$n$个位置作为球，得到结果$C(n + m - 1, n)$。
等价问题：
$$x_1 + x_2 + … + x_m = n \ x_i \geq 0, i=1,2…m$$

球无区别，盒子有区别（不允许有空盒）

结论：$F(m, n - m) = C(n - 1, m - 1)$。

等价问题：
$$x_1 + x_2 + … + x_m = n \ x_i > 0, i=1,2…m$$
每个$x_i, i=1,2,…m$都减一，将问题转化为：
$$x_1 + x_2 + … + x_m = n - m \ x_i \geq 0, i=1,2…m$$

球有区别，盒子有区别（不允许有空盒）

结论：$m!S_2(n, m)$。
转换思路：首先我们假设盒子没有区别，对应的模型就是第二类的stirling数。当盒子有区别时，盒子的每一种不同的排列方式都对最后的结果有贡献，易得结论。
容斥原理：$p_1$是1号盒子为空，$p_2$是$2$号盒子为空，$…$，$p_m$是$m$号盒子为空，对应的集合为$A_1, A_2, …, A_m$。用容斥原理的公式求解。

球有区别，盒子有区别（允许有空盒）

结论：$m^n$。
每一个球都可以放在任何一个盒子中，有$m$种选择，最后的结果就是$m^n$。

浮点数的表示

发表于 2020-03-21 更新于 2020-11-23 分类于 日常学习

浮点数

浮点表示对形如$V=x\times2^y$的有理数进行编码。他对执行涉及非常大的数字以及非常接近0的数字，更普遍地作为实数运算的近似值的计算是很有用的。

二进制小数

理解浮点数的第一步是考虑含有小数值的二进制数字。十进制表示法使用如下形式的表示：
$$d_md_{m-1}···d_1d_0d_{-1}d_{-2}···d_{-n}$$
其中每个十进制数$d_i$的取值范围是$0~9$，这个表达描述的数值$d$的定义如下：
$$d=\sum_{i=-n}^m10^i\times d_i$$
数字权的定义域十进制小数点符号相关，小数点左边的数字的权是10的正幂，得到整数值，而小数点右边的数字的权是10的负幂，得到小数值。

那么，对于二进制的一个数字，
$$b_mb_{m-1}···b_1b_0b_{-1}b_{-2}···b_{-n}$$
他的每一位的取值范围是0和1，这种表示方法表示的数$b$定义如下：
$$b=\sum_{i=-n}^m2^i\times b_i$$
从这个等式里面可以容易的看出来，二进制小数点向左移动一位相当于这个数被2除。同理向右移动一位的时候相当于将该数乘2。

假设我们只考虑有限长度的编码，那么十进制法不能准确地表达像$\frac{1}{3}$这样的数。同理，小数的二进制表示法只能表示那些能够写成$x\times 2^y$的数。

IEEE浮点表示

IEEE浮点标准用$V=(-1)^s\times M\times 2^E$的形式来表示一个数：

  • 符号（sign） $s$决定这个数是符数还是正数，对于数值0的符号位解释作为特殊情况处理。
  • 尾数（significand） M是一个二进制小数，他的范围是$1$$2-\epsilon$，或者是$0$$1-\epsilon$。
  • 阶码（exponent） $E$的最用是对浮点数加权，这个权重是2的$E$次幂（可能是负数）。
  • 一个单独的符号位$s$直接编码符号$s$。
  • k位的阶码字段$exp=e_{k-1}···e_1e_0$编码阶码$E$。
  • n位小数字段$frac=f_{n-1}···f_1f_0$编码尾数$M$，编码出来的值依赖于解码字段的值是否等于0。

在单精度浮点格式中，$s$、$exp$和$frac$字段分别为1位、$k=8$位和$n=23$位。在双精度浮点格式中分别为1位、11位和52位。

给定位标识，根据$exp$的值，被编码的值可以分为三种不同的情况，图2-33说明了对单精度格式的情况。

  • 情况1：规格化的值。当$exp$的位模式不全为0，也不全为1的时候，都属于这种情况。在这种情况中，阶码字段被解释为以偏置（biased）形式标识的有符号整数。也就是说，阶码的值是$E=e-Bias$，其中e是无符号数，而$Bias$是一个等于$2^{k-1}-1$的偏置值。因此产生了指数的取值范围，对于单精度来说是-126+127，对于双精度是-1022+1023。（不能全0或全1）小数字段$frac$被解释为描述小数值$f$，其中$0\le f<1$，尾数定义为$M=1+f$，有时这种方式也被称为隐含的以1开头的（implied leading 1）表示。
  • 情况2：非规格化的值。当阶码域全为0时，所表示的数是非规格化形式。这种情况下，**阶码值是$E=1-Bias$**（非规格化的数向规格化的数的平滑过渡），而尾数的值是$M=f$，不包含隐含的开头的1。非规格化的数有两个用途，首先，他们提供了一种表示数值0的方法。因为在规格化的数中，$M\ge1$，因此我们就不能表示0。
  • 情况3：特殊值。当阶码全为1的时候出现。当小数域全为0的时候，得到的值表示无穷。当符号位$s=0$时是$+\infin$，当符号位$s=1$时是$-\infin$。当小数域非0的时候，结果值被称为$NaN$。

C语言基本概念

编译和链接

C语言源代码转化成机器指令的三个步骤：

  • 预处理。把定义的宏以及include的内容直接搞过来。
  • 编译。编译器把预处理过的程序翻译成机器指令，生成.o结尾的文件。但是还不能运行。
  • 链接。链接器（linker）把编译器产生的目标代码和其他附加代码整合到一起，产生可以执行的程序。

一般以#开头的称为指令。函数里面叫语句。

C语言区分大小写。
不能用关键字作标识符。

格式化的输入/输出

讲了讲printf，scanf，提了一下gets和puts。

表达式

关键是优先级。

由实现定义（implementation-defined）：由具体的平台的实现决定。

二元运算符是左结合的（从左往右运算），一元运算符是右结合的（从右往左，如+-i = +(-i)）。

SQL查询语言概览

SQL的早期版本是IBM开发的，最初被叫做Sequel。Sequel语言发展到现在名称已经变为SQL（Structure Query Language）。
最早的SQL标准是SQL-86。1989年ANSI发布了一个SQL的扩充标准：SQL-89。接着又有了下一个版本SQL-92的标准，后面还有很多标准。
SQL有以下几个部分：

  • 数据定义语言（Data-Definition Language）：SQL DDL提供定义关系模式、删除关系模式以及修改关系模式的命令。
  • 数据操作语言（Data-Manipulation Language）: SQL DML提供从数据库中查询信息，以及在数据库中插入元组、删除元组、修改元组的能力。
  • 完整性（integrity）：SQL DDL包括定义完整性约束的命令，保存在数据库中的数据必须满足所定义的完整性约束。
  • 视图定义（view definition）：SQL DDL包括定义视图的命令。
  • 事务控制（transaction control）：SQL包括定义事务开始和结束的命令。
  • 嵌入式SQL和动态SQL（embedded SQL and dynamic SQL）：嵌入式和动态SQL定义SQL语句如何其纳入到通用编程语言，如C、C++和Java。
  • 授权（authorization）：SQL DDL包括定义对关系和视图的访问权限的命令。

关系数据库的结构

关系数据库由表（table）的集合构成，每个表有唯一的名字。
一般来说，表的一行代表了一组值之间的一种联系。一个表就是这种联系的一个集合，所以表的概念和数学上的关系是密切相关的，这也是关系数据模型的由来。

数学术语中，元组（tuple）是一组值的序列。
在关系模型的术语中，关系（relation）用来指代表，元组（tuple）用来指代行。属性（attribute）指代的是表中的列。


在表2-1中，instructor关系有四个属性：ID、name、dept_name和salary。

关系实例（relation instance）：表示一个关系的特定实例，也就是包含一组特定的行。图2-1所示的instructor的实例有12个元组，对应12个教师。

关系是一个元组集合，在集合中，元组的顺序是无关紧要的。

属性的域（domain）：对于每个属性，都存在一个允许取值的集合，称为这个属性的域。（某属性取值的集合称为域）
要求对于所有关系r而言，其所有属性的域都是原子的（atomic）。（也可以不是原子的，反之目前要求是原子的）

空（NULL）：特殊值，表示值未知或者不存在。应该尽量避免。

数据库模式

数据库模式（database schema）：数据库的逻辑设计。
数据库实例（database instance）：给定时刻数据库中数据的一个快照。
两者的区别就是关系对应于程序设计语言中变量的概念，关系模式（relation schema）对应于程序设计语言中类型定义的概念。关系实例的概念就是程序设计语言中变量的值的概念。变量的值可能随时间发生变化；关系的模式是不常变化的。

码

必须有一种能够区分给定关系的不同元素的方法，这用他们的属性来表明。也就是说，一个元组的属性值必须是能够唯一区分元组的。换句话说，一个关系中没有两个元组在所有的属性上的取值都相同。（真绕啊）

超码（superkey）：一个或多个属性的集合，这些属性的组合可以使我们在一个关系中唯一地标识一个元组。
比如说一个人的身份证号可以把这个人区分出来，因此身份证号是一个超码，反之名字不是，因为重名的人千千万。

形式化（不说人话）的描述就是：

$$设R表示关系r模式中的属性集合。\ 如果说R的一个子集K是r的一个超码，\ 则限制了关系R中任意两个不同元组不会在K的所有属性上取值完全相等，\ 即如果t_1和t_2在r中且t_1\neq t_2，则t_1.K\neq t_2.K。$$

超码中也可能包含无关紧要的属性。和线代中极大线性无关组与任何一个其他的向量裹在一起也是一个线性无关组是一个道理。如果一个有一个超码，这个超码的真子集一定不是超码，它的任意超集也一定是超码。这样的最小超码称为候选码（candidate key）。
候选码一定要是最小那个集。也就是说当{attr_1, attr_2}中有一个是超码的时候，那么这个属性的集合就不能当作候选码。

外码（foreign key）或参照关系（referencing relation）：一个关系模式$r_1$可能在它的属性中包括了另一个关系模式$r_2$的主码。这个属性$r_1$被称为参照$r_2$的外码。

关系查询语句

查询语句（query language）：用户用来从数据库中请求获取信息的语言。

过程化语言（procedural language）：用户直到系统对数据库执行一系列操作以计算出所需的结果。

非过程化语言（nonprocedural language）：用户只需要描述所需信息，而不用给出获取该信息的具体过程。

实际使用的查询语句包含两种成分。

关系运算

所有的过程化关系查询语言都提供了一组运算，要么施加于单个关系上，要么施加于一对关系上。这些运算具有一个很好的性质：运算的结果总时单个的关系。这样我们可以用模块化的方式来组合几种这样的运算。因为关系查询的结果本身也是关系，所以关系运算可以施加到查询结果上。

常用的关系运算有：

  • 从单个关系中选出满足一些特定谓词的特殊元组。
  • 从一个关系中选出特定的属性。
  • 连接运算，把分别来自两个关系的元组对合并成单个元组。
  • 笛卡尔积运算。
  • 关系是集合，因此也可以并交差。

碰到的问题

代码也很简单，不过我是用的C语言，因此不能直接将已经为空的链表从arraylist那种链表中直接remove，所以多了中间几行把非空的链表换到已经成空链表的位置，并且使listsSize减一。

具体的问题其实是条件判断的顺序是从左到右的，这里先判断lists[i] != NULL，很显然当i = listsSize - 1且循环结束了过后i++导致i = ListsSize，这里就越界了。
最开始写代码的时候会犯这种错误，结果写了这么久代码仍然会犯这种错误…浪费总时长不少于5个小时，太亏了。
